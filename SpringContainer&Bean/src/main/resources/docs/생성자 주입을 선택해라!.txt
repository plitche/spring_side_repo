* 생성자 주입을 선택해라!
    - 과거에는 수정자 주입과 필드 주입을 많이 사용했지만, 최근에는 스프링을 포함한 DI 프레임워크 대부분이 생성자 주입을 권장한다.

    * 이유
        - 불변
            : 대부분의 의존관계 주입은 한번 일어나면 애플리케이션 종료시점까지 의존관계를 변경할 일이 없다.
            : 오히려 종료 전까지 변하면 안된다.
            : 수정자 주입을 사용하면 setXxx 메서드를 public으로 열어두어야 한다.
            : 누군가 실수로 변경할 수도 있고 변경하면 안되는 메서드를 열어두는 것은 좋은 설계 방법이 안디ㅏ.
            : 생성자 주입은 객체를 생서할 때 1번만 호출되므로 이후에 호출되는 일이 없다.

        - 누락
            : 프레임워크 없이 순수한 자바 코드 단위 테스트하는 경우에 @Autowired가 프레임워크 안에서
            동작할 때는 의존관계가 없으면 오류가 발생하지만, 지금은 프레임워크 없이 순수한 자바 코드로만 단위 테스트를 수행하고 있다.
